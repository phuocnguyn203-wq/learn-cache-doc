Why bother caching?

Because doing the same expensive work over and over is a waste — of time, of money, and of your server's patience.

⚡

Faster responses

Serve a stored answer in microseconds instead of redoing a slow computation or database query every single time.

💸

Lower cost

Fewer database hits and fewer calls to paid third-party APIs means smaller infrastructure bills.

📈

Handles more traffic

A cache absorbs repeated requests so your database or upstream service doesn't fall over during a spike.

🙂

Happier users

Nobody enjoys a spinner. Caching is one of the highest-leverage ways to make an app feel instant.

Every line, explained

No black boxes. Each lesson builds its code up piece by piece, in plain FastAPI.

main.py 
cache = {}

@app.get("/weather/{city}")
def get_weather(city: str):
    if city in cache:
        return cache[city]  # cache hit — skip the slow work

    time.sleep(3)  # pretend this is a slow call to a weather service
    weather = {"city": city, "temperature": 24}
    cache[city] = weather  # cache miss — save it for next time
    return weather
