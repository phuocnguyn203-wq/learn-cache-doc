Your First Cache

Let's build the smallest possible cache there is: a single Python dictionary. We'll go through every line of code — assuming nothing — and by the end you'll have a real endpoint that skips repeated slow work.

The problem, in code

Imagine we're building a weather app. Whenever someone asks for the weather in a city, our server calls a weather service — and pretend that service is slow, taking 3 whole seconds to answer. Here's that endpoint:

main.py
import time
from fastapi import FastAPI

app = FastAPI()

@app.get("/weather/{city}")
def get_weather(city: str):
    time.sleep(3)  # pretend this is a slow call to a weather service
    return {"city": city, "temperature": 24}

Run this and visit it a few times — even for the exact same city, every single visit waits the full 3 seconds. The server never remembers anything. Let's give it a memory.

A cache is just a box with labeled drawers

Picture a box of little drawers. Each drawer has a label — like "Paris" or "Tokyo" — and inside sits a piece of paper with the answer. Python already has exactly this kind of box built in. It's called a dictionary (dict for short):

a plain Python dict, no FastAPI yet
box = {}            # an empty box, no drawers yet
box["Paris"] = 24   # put 24 in a new drawer labeled "Paris"
print(box["Paris"])  # open the "Paris" drawer and read what's inside -> 24

One more trick we'll use constantly: "Paris" in box asks "does a drawer with this label exist?" — without needing to open it. That's exactly what a cache needs to do first: check if an answer is already sitting in a drawer before doing any slow work.

Making our endpoint use the box

1

Create the box — but only once

We add one line, outside the function, near the top of the file:

main.py
cache = {}
🧒 Why does it matter where this line lives?

If we wrote cache = {} inside the function, Python would build a brand-new, empty box every single time someone visited — throwing away whatever we'd saved a moment earlier. Writing it outside the function means it's created once, when the server starts up, and it quietly stays alive in memory for as long as the server keeps running.

2

Check the box first — a cache hit

Before doing any slow work, look in the box:

main.py
if city in cache:
    return cache[city]

If a drawer with this city's name already exists, we skip everything else and hand back what's inside immediately. This is exactly what a cache hit means.

3

If it's not there, do the slow work — then save it

This is a cache miss: the drawer was empty, so we do the real work and fill it in before we leave, so next time is a hit.

main.py
time.sleep(3)
weather = {"city": city, "temperature": 24}
cache[city] = weather
return weather

The full endpoint

Put together, here's the whole file:

main.py
import time
from fastapi import FastAPI

app = FastAPI()
cache = {}

@app.get("/weather/{city}")
def get_weather(city: str):
    if city in cache:
        return cache[city]

    time.sleep(3)
    weather = {"city": city, "temperature": 24}
    cache[city] = weather
    return weather

Try it yourself

Start the server (uvicorn main:app --reload) and request the same city twice:

Terminal
curl http://localhost:8000/weather/paris
# ~3 seconds later: {"city":"paris","temperature":24}

curl http://localhost:8000/weather/paris
# instantly: {"city":"paris","temperature":24}

The second call comes back immediately. That's your cache working.

What did we just build?

No matter how many times someone asks for the same city, the slow work happens only once. Check first, do the work and save it only if you have to — this exact shape is something you'll see again and again in caching, even though we're not naming it yet.

⚠️ What this cache doesn't do yet
  • It treats every different input as a totally separate drawer — but what if two different questions should share an answer, or one question needs more than just a city to be correct? → Cache Keys
  • It never forgets. If the real weather changes, we keep serving today's number forever. → TTL
  • It has no size limit — cache a million cities and it grows forever. → Eviction
  • If someone corrects the data by hand, the cache doesn't know. → Invalidation

The next four lessons fix each of these, one small piece at a time.
