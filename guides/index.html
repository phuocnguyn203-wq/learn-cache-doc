Guides

The Learn section builds caching up from zero, one idea at a time. These guides start where that leaves off: standard terminology, real libraries, and the trade-offs a professional team actually weighs when caching goes into production. Every example is real, runnable Node.js (Express) code.

🧭 Coming from Learn?

These guides assume you're already comfortable with cache hits/misses, TTL, cache keys, eviction, and invalidation. If any of those feel unfamiliar, start with What is Caching? first — the guides won't re-explain them.

Prerequisites

  • Comfortable with the vocabulary and patterns from Learn: cache hit/miss, TTL, keys, eviction, invalidation.
  • A working Node.js/Express project, or enough comfort with either to follow along.
  • Redis (local install or Docker) is needed for most examples: docker run -p 6379:6379 redis.
  • These guides don't build on each other in strict order — read whichever one matches the problem in front of you.
